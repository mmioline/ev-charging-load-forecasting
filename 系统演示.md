# 系统核心业务场景演练与验收说明

本篇文档面向测试验收环节，旨在演示系统在多物理微服务容器集群下，核心业务链路的完整流转。所有演示步骤均已通过 `api_pytest` 自动化测试集进行置信度卡点验证。

---

## 一、 测试演练前置准备

在进行核心业务场景验证前，请确保本地已配置好完整的容器集群环境。

### 步骤 1：一键重置并拉起微服务集群

在项目根目录下执行以下命令，强力清理旧容器环境并重新构建，确保测试环境的数据隔离（幂等性）：

```bash
docker compose down -v --remove-orphans
docker compose up -d --build
```

### 步骤 2：自动化注入业务底座测试数据

若需要一次性完成环境重置、账号创建、站点创建、历史记录注入、模型训练与自动化回归，可执行：

```bash
chmod +x ./run_pipeline.sh
./run_pipeline.sh
```

## 二、 核心业务场景验收演练

### 场景一：端到端用户鉴权与路由拦截 (安全边界演练)

- 业务预期：
  未注册或 Token 过期的客户端请求应当被服务安全层精准拦截，返回 401 Unauthorized；合法登录用户发放标准 JWT 令牌，并放行后续业务请求。

- 验收步骤：
  1. 使用 Postman 或 curl 模拟客户端，在未携带 Header 凭证时，直接发起请求：

     ```text
     GET http://localhost:28011/stations/status/available
     ```

     （查询空闲充电站）。

  2. 系统表现：接口拦截请求，拒绝返回站点数据，状态码显示 401 Unauthorized。

  3. 客户端向用户服务发起表单请求：

     ```text
     POST http://localhost:28010/login
     ```

     输入预设测试账号。

  4. 系统表现：user_service 校验成功，返回具有 30 分钟时效性的标准 JWT 鉴权令牌。

  5. 客户端将令牌注入请求头（Authorization: Bearer <TOKEN>）重新请求场景 1 的查询接口。

  6. 系统表现：服务放行，成功获取 JSON 格式的空闲站点列表。

### 场景二：充电桩生命周期状态机流转 (核心业务流验证)

- 业务预期：
  模拟实车接入充电桩后，站点状态、充电流水创建以及高频并发读写下的数据状态机强一致性变化。

- 验收步骤：

  1. 初始状态检查：
     访问站点监控服务，目标充电桩在 station_service 中注册状态显示为 空闲 (Free)。

  2. 实车接入演练：
     客户端发送同步请求到 charging_service，触发创建一条全新的充电流水记录。

  3. 系统实时原子流转表现：

     - 后端 charging_service 成功落库持久化，生成流水单号。

     - 自动触发内部路由事件，将 station_service 中对应的目标桩状态强行锁死为 充电中 (Charging)。

  4. 边界异常验证：
     在目标桩处于“充电中”状态时，尝试并发注入第二笔充电请求。

  5. 系统表现：
     接口精准拦截，业务逻辑判定失败，抛出错误提示“站点当前状态为「充电中」，无法发起充电”，成功规避并发脏数据写入。

### 场景三：基于时序大数据的负荷智能推理演练 (数据价值校验)

- 业务预期：
  模拟监控大屏或智能调度中心定时拉取未来时间段内站点的空闲度与负荷趋势，验证 AI 时序推理能力，贴合新能源充电适配与大数据分析场景。

- 验收演练步骤：

  1. 系统定时任务或人工请求向预测微服务触发请求：

     ```text
     GET http://localhost:28013/predict/{station_id}
     ```

  2. 全链路通信与系统表现：

     - forecasting_service 实时响应，激活底层训练就绪的 PyTorch/LSTM 模型。

     - 模型自动读取该站点前置的充电时序历史特征，执行多维矩阵推理。

  3. 验收结果：
     接口以标准规范的 JSON 格式输出 predicted_load_kwh（预测未来负荷值）与可用性概率，成功为站点错峰导流、整车充电适配提供数字化支撑。
