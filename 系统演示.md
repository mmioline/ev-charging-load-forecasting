## 一、环境确认

当前系统对外只暴露 4 个业务端口：

```text
18010  用户服务
18011  站点服务
18012  充电记录服务
18013  预测服务
18080  前端页面
```

MySQL 和 Redis 只在 Docker 内部网络使用，`docker compose ps` 中看到 `3306/tcp`、`6379/tcp` 是容器内部端口，不是宿主机暴露端口。

```bash
docker compose ps

curl -sS http://localhost:18010/health | python3 -m json.tool
```

如果服务状态不干净，可以重置：

```bash
docker compose down -v --remove-orphans
docker compose up -d --build --force-recreate
docker compose ps
```

## 三、获取双账号 Token

```bash
ALICE_TOKEN=$(curl -sS -X POST http://localhost:18010/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'username=alice&password=alice1234' \
  | python3 -c 'import sys,json; print(json.load(sys.stdin)["access_token"])')

BOB_TOKEN=$(curl -sS -X POST http://localhost:18010/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'username=bob&password=bob1234' \
  | python3 -c 'import sys,json; print(json.load(sys.stdin)["access_token"])')

echo "Alice Token: ${ALICE_TOKEN:0:30}..."
echo "Bob Token: ${BOB_TOKEN:0:30}..."
```

## 四、创建三个充电站

不要假设站点 ID 一定是 `1、2、3`。下面命令会自动保存真实 ID。

```bash
A_ID=$(curl -sS -X POST http://localhost:18011/stations \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "科技园A站", "location": "深南大道1号", "capacity": 60.0, "slots": 6}' \
  | python3 -c 'import sys,json; print(json.load(sys.stdin)["id"])')

B_ID=$(curl -sS -X POST http://localhost:18011/stations \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "总部基地B站", "location": "科苑路88号", "capacity": 160.0, "slots": 16}' \
  | python3 -c 'import sys,json; print(json.load(sys.stdin)["id"])')

C_ID=$(curl -sS -X POST http://localhost:18011/stations \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "老城区C站", "location": "人民南路99号", "capacity": 100.0, "slots": 10}' \
  | python3 -c 'import sys,json; print(json.load(sys.stdin)["id"])')

echo "A_ID=$A_ID B_ID=$B_ID C_ID=$C_ID"
```

确认站点：

```bash
curl -sS http://localhost:18011/stations \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

## 五、模拟 C 站超时异常

```bash
curl -sS -X PUT http://localhost:18011/stations/$C_ID/status \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "充电中"}' | python3 -m json.tool

docker exec intellishop_db mysql -uroot -proot intelli_shop \
  -e "UPDATE stations SET last_updated_at = DATE_SUB(NOW(), INTERVAL 25 HOUR) WHERE id = $C_ID;"

docker exec intellishop_db mysql -uroot -proot intelli_shop \
  -e "SELECT id, name, status, last_updated_at FROM stations WHERE id = $C_ID;"
```

触发自动判定：

```bash
curl -sS http://localhost:18011/stations/$C_ID/status \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

预期：C 站状态自动变为 `异常`。

## 六、注入 100 条历史充电记录

`seed_data.py` 默认使用 `18010/18012`，并向 A、B 两站注入数据：

```bash
source venv/bin/activate
SEED_STATION_IDS="$A_ID,$B_ID" python services/seed_data.py
```

把记录时间分散到过去 30 天：

```bash
docker exec intellishop_db mysql -uroot -proot intelli_shop \
  -e "SET @row := 0; UPDATE charging_records SET created_at = DATE_SUB(NOW(), INTERVAL ((@row := @row + 1) % 30) DAY);"
```

验证记录数量：

```bash
docker exec intellishop_db mysql -uroot -proot intelli_shop \
  -e "SELECT station_id, COUNT(*) AS cnt FROM charging_records GROUP BY station_id;"
```

## 八、正式演示

先打开前端页面：

```text
http://localhost:18080
```

如果浏览器无法打开，先确认前端容器端口：

```bash
docker compose ps frontend
docker port frontend
```

正常应看到：

```text
8080/tcp -> 0.0.0.0:18080
8080/tcp -> [::]:18080
```

如果在 Windows 浏览器中访问 `localhost:18080` 失败，可以在 PowerShell 中查看 WSL 地址：

```powershell
wsl hostname -I
```

然后用返回的第一个 IP 访问，例如：

```text
http://<WSL_IP>:18080
```

备用方案：直接打开静态文件：

```text
\\wsl.localhost/Ubuntu-24.04/home/shikari/myshop/frontend/index.html
```

前端按钮仍然会调用 `18010-18013` 四个后端接口。

### 8.1 Alice 完整业务链

查询初始站点状态：

```bash
curl -sS http://localhost:18011/stations \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

查询空闲站点，演示 C 站被异常剔除：

```bash
curl -sS http://localhost:18011/stations/status/available \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

查看 C 站 Redis 缓存：

```bash
docker exec ev_station_redis redis-cli GET "station:status:$C_ID"
```

Alice 向 A 站发起充电：

```bash
ALICE_RECORD_ID=$(curl -sS -X POST http://localhost:18012/charging \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"station_id\": $A_ID, \"duration_minutes\": 45, \"kwh_consumed\": 22.5}" \
  | python3 -c 'import sys,json; data=json.load(sys.stdin); print(data["id"])')

echo "ALICE_RECORD_ID=$ALICE_RECORD_ID"
```

验证 A 站状态变为 `充电中`，并从空闲列表移除：

```bash
curl -sS http://localhost:18011/stations/$A_ID/status \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool

curl -sS http://localhost:18011/stations/status/available \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

Alice 向 B 站发起第二次充电：

```bash
curl -sS -X POST http://localhost:18012/charging \
  -H "Authorization: Bearer $ALICE_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"station_id\": $B_ID, \"duration_minutes\": 30, \"kwh_consumed\": 15.0}" \
  | python3 -m json.tool
```

预测 A 站未来 1 小时空闲概率：

```bash
curl -sS http://localhost:18013/predict/$A_ID | python3 -m json.tool
```

### 8.2 Bob 鉴权与订单隔离

Alice 查询自己的身份：

```bash
curl -sS http://localhost:18010/users/me \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

Bob 查询自己的身份：

```bash
curl -sS http://localhost:18010/users/me \
  -H "Authorization: Bearer $BOB_TOKEN" | python3 -m json.tool
```

Alice 能查询自己的充电记录：

```bash
curl -sS http://localhost:18012/charging/$ALICE_RECORD_ID \
  -H "Authorization: Bearer $ALICE_TOKEN" | python3 -m json.tool
```

Bob 查询同一条 Alice 记录会被拒绝：

```bash
curl -i -sS http://localhost:18012/charging/$ALICE_RECORD_ID \
  -H "Authorization: Bearer $BOB_TOKEN"
```

预期返回 `404 Charging record not found`，避免泄露 Alice 订单存在性。

无 Token 访问受保护接口：

```bash
curl -i -sS http://localhost:18011/stations
```

伪造 Token 被拒绝：

```bash
curl -i -sS http://localhost:18011/stations \
  -H "Authorization: Bearer this.is.a.fake.token"
```
